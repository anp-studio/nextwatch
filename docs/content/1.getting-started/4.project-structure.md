## Project Layout

The Movie Recommender project has three main directories:

```text
movie-recommender/
├── app/                 # Nuxt 4 full-stack app (where most work happens)
├── database/            # SQLite schema and database for local development
├── docs/                # Documentation site
├── README.md
└── LICENSE
```

## App Directory

The main app is a Nuxt 4 application with Vue and Tailwind CSS. All active development happens here.

```text
app/
├── app/
│   ├── app.vue
│   ├── pages/
│   ├── components/
│   ├── composables/
│   ├── layouts/
│   ├── plugins/
│   └── types/
├── server/
│   ├── api/
│   │   ├── tmdb/        # TMDB proxy
│   │   ├── watched/     # Watched movies API
│   │   ├── gemini/      # AI recommendations
│   │   ├── movies/      # Movie search
│   │   └── admin/       # Admin endpoints
│   ├── tasks/           # Scheduled tasks
│   └── utils/           # Server utilities
├── public/
├── nuxt.config.ts
├── tailwind.config.js
├── tsconfig.json
└── package.json
```

### Key Areas

- **Pages** (`app/pages/`) - Home, search, watched, profile, chat, reset-password
- **Components** (`app/components/`) - Reusable UI components
- **Composables** (`app/composables/`) - Shared logic for auth, watched state, details, and filters
- **Server API** (`server/api/`) - Backend endpoints for recommendations, TMDB proxy, watched movies, and admin import

## Database Directory

Contains the SQLite database and schema:

```text
database/
├── local.db            # SQLite database with movie data
└── schema.sql          # Database schema
```

The database is populated daily with movie data from TMDB.

## Documentation Directory

The docs site (what you're reading now):

```text
docs/
├── content/            # Markdown pages
├── public/             # Static assets
├── nuxt.config.ts
└── package.json
```

## What to Know

- All commands (`npm run dev`, `npm install`, etc.) are run from the `app/` directory
- The server runs on `http://localhost:3000`
- The docs site is separate — it has its own `package.json` and dev server
- The database is stored as a local SQLite file in `database/local.db`
