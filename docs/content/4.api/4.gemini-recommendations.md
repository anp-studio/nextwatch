# Gemini Recommendations API

`/api/gemini/recommend` uses Google Gemini to generate movie recommendations based on the user's watch history.

## Auth

- Requires `Authorization: Bearer <supabase_access_token>`
- User is resolved server-side from the token
- Requests without a valid token return `401 Unauthorized`

## How It Works

1. Authenticates the user via Supabase JWT token
2. Fetches the user's watched movies from the `watched_movies` table
3. Merges with any locally-tracked movies from the request body (deduped by tmdbId)
4. Sends the merged watch history to Gemini with structured JSON output schema
5. Returns 10 movie recommendations (maximum is 50)

## Endpoints

### `POST /api/gemini/recommend`

Generates 10 movie recommendations based on watched movies.

#### Request body (optional)

Include locally-tracked movies not yet synced to Supabase:

```json
{
  "movies": [
    { "id": 550, "title": "Fight Club", "year": 1999 },
    { "id": 278, "title": "The Shawshank Redemption", "year": 1994 }
  ]
}
```

**Note:** `id` refers to the movie's TMDB ID. The request body is optional — if omitted, only Supabase watch history is used.

#### Response

```json
{
  "recommendations": [
    { "name": "Interstellar", "year": 2014 },
    { "name": "The Dark Knight", "year": 2008 },
    { "name": "Inception", "year": 2010 },
    { "name": "Pulp Fiction", "year": 1994 },
    { "name": "Forrest Gump", "year": 1994 },
    { "name": "The Matrix", "year": 1999 },
    { "name": "Goodfellas", "year": 1990 },
    { "name": "The Green Mile", "year": 1999 },
    { "name": "Se7en", "year": 1995 },
    { "name": "The Silence of the Lambs", "year": 1991 }
  ]
}
```

#### Behavior

- Fetches user's Supabase watch history
- Merges with body movies (if provided), deduped by TMDB ID
- Returns `400` if no watched movies exist
- Sends merged list to Gemini with structured output schema (name + year)
- Validates response and returns exactly 10 recommendations

## Error Codes

- `400` no watched movies found
- `401` missing or invalid bearer token
- `500` Supabase query failure or Gemini API key not configured
- `502` Gemini returned invalid JSON or response did not match schema

## Integration with Frontend

Use the `useMovies()` composable to send pending watched movies:

```typescript
const { pendingWatchedMovies } = useMovies()
const { data } = await $fetch('/api/gemini/recommend', {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${session.access_token}`,
  },
  body: {
    movies: pendingWatchedMovies.value,
  },
})

console.log(data.recommendations) // Array<{ name: string, year: number }>
```

## Environment

Requires `NUXT_GEMINI_API_KEY` to be set in `.env`. If missing, the endpoint returns `500`.
