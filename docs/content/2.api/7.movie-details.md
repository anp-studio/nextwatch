# Movie Details API

The Movie Details endpoint fetches enriched metadata for a single movie by its TMDB ID. Results are cached server-side in the `movies_metadata` table for **3 months** to avoid redundant TMDB API calls.

## Endpoint

```text
GET /api/movies/:id
```

## How It Works

1. Client requests `/api/movies/:id` with a TMDB movie ID
2. Server checks `movies_metadata` for a fresh cached record (less than 3 months old)
3. **Cache hit** â€” returns the cached record immediately, no TMDB call made
4. **Cache miss** â€” fetches from TMDB (`/movie/:id?append_to_response=credits,videos`), returns the response, and writes to cache in the background (non-blocking)

## Example Usage

```js
const response = await fetch('/api/movies/550')
const movie = await response.json()
```

## Path Parameter

| Parameter | Type | Description |
|-----------|------|-------------|
| `id` | `number` | TMDB movie ID (positive integer) |

Returns `400` if the ID is missing or not a positive integer.

## Response Format

```json
{
  "id": 550,
  "title": "Fight Club",
  "poster": "https://image.tmdb.org/t/p/w500/path/to/poster.jpg",
  "rating": 8.4,
  "year": 1999,
  "duration": "2h 19m",
  "genres": ["Drama", "Thriller"],
  "actors": ["Brad Pitt", "Edward Norton", "Helena Bonham Carter", "Meat Loaf", "Zach Grenier"],
  "description": "A ticking-time-bomb insomniac...",
  "trailer": "https://www.youtube.com/watch?v=SUXWAEX2jlg"
}
```

| Field | Type | Notes |
|-------|------|-------|
| `id` | `number` | TMDB ID |
| `title` | `string` | Movie title |
| `poster` | `string` | Full image URL (`w500`). Empty string if no poster. |
| `rating` | `number` | `vote_average` rounded to 1 decimal place |
| `year` | `number` | Release year parsed from `release_date` |
| `duration` | `string` | Formatted as `2h 19m`. `"N/A"` if runtime is unknown. |
| `genres` | `string[]` | Genre names |
| `actors` | `string[]` | Top 5 cast members |
| `description` | `string` | Movie overview |
| `trailer` | `string \| null` | YouTube URL of the most recent official trailer, or `null` |

## Caching Details

| Property | Value |
|----------|-------|
| Cache store | `movies_metadata` table (LibSQL/SQLite) |
| TTL | 3 months (90 days) |
| Cache key | `tmdb_id` |
| Write strategy | Non-blocking â€” cache write does not delay the response |
| Trailer selection | Most recent official YouTube trailer (`type = "Trailer"`, `official = true`) |

Cache freshness is checked via the `cached_at` Unix timestamp column. A record is considered stale if `cached_at < now - TTL`.

## Error Handling

| Status | Cause |
|--------|-------|
| `400` | `id` is missing or not a positive integer |
| `404` | TMDB returned no movie for that ID |
| `500` | TMDB API or database failure |
