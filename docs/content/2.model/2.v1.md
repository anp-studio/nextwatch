## Overview

V1 model is the foundation of the Movie Recommender system. It uses **content-based filtering** with cosine similarity to find the most similar movies and shows based on multiple features like genres, overview text, ratings, and production details.

### Characteristics

- **Speed**: Fast computation (\~10-30 seconds for 5K items)
- **Accuracy**: Good for Marvel movies
- **Penalties**: Quality penalties for very different ratings or untested films

---

## Algorithm Overview

### Feature Engineering

The model creates afeature vector for each movie/show combining 9 different features:

| Feature          | Type                  | Weight | Purpose                     |
| ---------------- | --------------------- | ------ | --------------------------- |
| **Genres**       | One-hot encoding      | 3.0x   | Direct genre matching       |
| **Overview**     | TF-IDF (500 features) | 2.0x   | Semantic content similarity |
| **Type**         | Binary (movie/show)   | 1.5x   | Prefer same media type      |
| **Rating**       | Normalized 0-1        | 5.0x   | Quality filtering           |
| **Popularity**   | Log-normalized        | 2.5x   | Popular films preference    |
| **Vote Count**   | Log-normalized        | 2.0x   | Prefer voted movies         |
| **Release Year** | Normalized 1900-2026  | 2.0x   | Era similarity              |
| **Budget Tier**  | 0-6 categories        | 4.0x   | Production scale            |
| **Revenue Tier** | 0-6 categories        | 2.0x   | Box office success          |

### Computation Steps

::steps
### Load Data

Load core movies and shows, combine datasets, rename columns for consistency, and filter only core items.

### Parse Features

Parse genres from string format and validate data integrity. Handle missing values and data type conversions.

### Create Feature Vectors

Create feature representations:

- Genre features (one-hot encoding)
- Overview features (TF-IDF vectorization with 500 features)
- Metadata features (rating, popularity, votes)
- Temporal features (release year normalized)
- Financial features (budget & revenue tiers)

Result: Feature matrix [5000 items × 500+ features]

### Compute Similarity

Apply cosine similarity between all feature vectors to measure how similar each pair is (0 = different, 1 = identical).

Result: [5000 × 5000] similarity matrix

### Apply Franchise Boost

Extract franchise names from titles (remove "Part II", "Episode 3", etc) and boost similarity for related items:

- Example: "The Matrix" + "The Matrix Reloaded" → +0.3 similarity boost
- Applied only within franchise groups

### Extract Top-N & Apply Penalties

For each item, extract 100 most similar and apply quality penalties:

- Rating mismatch (>1.5 points) → multiply by 0.5
- Low vote count (<100) → multiply by 0.3
- Budget mismatch (>10x or <0.1x) → multiply by 0.4

Save results as CSV: [id, similar\_id, similarity\_score]
::

## Key Functions

### `create_feature_vectors(df)`

Creates the feature matrix that powers similarity calculations.

**Process:**

1. **Genres**: One-hot encode genres (e.g., Action, Comedy, Drama)
2. **Overview**: TF-IDF with 500 max features, bigrams, English stop words removed
3. **Quality Metrics**: Normalize rating, popularity (log), vote count (log)
4. **Financial Features**: Budget and revenue organized into tiers (0-6)
5. **Temporal**: Release year normalized to [0, 1]

**Returns:** Feature matrix [N items × 500+ features]

```python
# Example usage
feature_matrix = create_feature_vectors(core_df)
print(feature_matrix.shape)
```

### `apply_sequel_boost(similarity_matrix, df)`

Increases similarity scores for franchises and sequels.

**Logic:**

1. Extract base franchise name from titles (remove "Part II", "Episode 3", etc.)
2. Group items by franchise name
3. Boost similarity within franchises: `min(1.0, score * 1.5 + 0.3)`

**Result:** Sequels score higher relative to unrelated films

```python
# Example: These pairs get boosted
# "Avengers" + "Avengers: Endgame" → 1.0x → 1.3x boost
# "The Matrix" + "The Matrix Reloaded" → 0.8x → 1.0x (capped at 1.0)
```

> Some pairs will have score of 1.0 which is too strong but that is fixed in future models

### `save_similarity_matrix(matrix, df)`

Extracts top 100 similar items per movie and applies quality penalties.

**Penalties Applied:**

- **Rating Mismatch** (>1.5 points): multiply by 0.5
- **Low Vote Count** (<100): multiply by 0.3
- **Budget Mismatch** (>10x or <0.1x): multiply by 0.4

**Output:** CSV file with columns `[id, similar_id, similarity_score]`

## Performance

> This results depend from machine to machine

```text
Loading & Processing:  ~2 seconds
Creating Features:     ~5 seconds
Cosine Similarity:     ~10-15 seconds  ← Bottleneck
Franchise Boost:       <1 second
Penalties & Save:      ~2 seconds
─────────────────────────────────
Total Time:            ~20-25 seconds
```

**Optimization Notes:**

- Uses numpy for vectorized operations
- Only applies penalties to top 250 candidates (not all 6000)
- Log normalization reduces outlier impact
- Tier-based features preserve meaningful differences

## Quality Checks

### Data Validation

- Missing ID handling - skips items with null ID
- Missing overview - fills with empty string
- Missing ratings - defaults to 5.0
- Floating point errors - clips scores to [0, 1]

### Penalty Application

The model applies quality penalties to prevent bad recommendations:

- **Rating difference** - Don't recommend 10/10 films to fans of 6/10 films
- **Low vote count** - Untested films are less reliable
- **Budget mismatch** - $200M blockbuster ≠ $2M indie film

These penalties multiply the score but never eliminate it completely.

## Limitations

| Text similarity bias | Anime films cluster together heavily   | Use weighted features (applied)  |
