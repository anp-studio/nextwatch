# Rate Limiting

TMDB and recommendation requests are rate-limited using [Upstash Ratelimit](https://upstash.com/docs/redis/sdks/ratelimit-ts/overview) with Upstash Redis.

::callout{type="info"}
These limits protect upstream APIs and prevent abuse. Clients should handle `429` responses gracefully.
::

## Limits

| Limiter                 | Limit       | Window   | Key       |
| ----------------------- | ----------- | -------- | --------- |
| `tmdbLimiter`           | 40 requests | 1 second | Client IP |
| `recommendationLimiter` | 20 requests | 1 day    | Authenticated user ID |

Both limiters use **sliding window**

## Response Headers

Every response that goes through `fetchTmdb` includes the following headers so clients can track remaining quota:

| Header                  | Description                              |
| ----------------------- | ---------------------------------------- |
| `X-RateLimit-Limit`     | Total requests allowed in the window     |
| `X-RateLimit-Remaining` | Requests remaining in the current window |
| `X-RateLimit-Reset`     | Timestamp (ms) when the window resets    |

### Example

```http
HTTP/1.1 200 OK
X-RateLimit-Limit: 40
X-RateLimit-Remaining: 37
X-RateLimit-Reset: 1743284610000
```

## Rate Limit Exceeded

When the limit is exceeded the server responds with `429 Too Many Requests`:

```json
{
  "statusCode": 429,
  "message": "Rate limit exceeded for TMDB API. Please try again later."
}
```

::callout{type="warning"}
Do not auto-retry `429` in a tight loop. Respect `X-RateLimit-Reset` before reattempting.
::

## IP Resolution

The client IP is resolved via `getRequestIP(event, { xForwardedFor: true })` which reads the `X-Forwarded-For` header. Falls back to `'anonymous'` if no IP can be determined.

## Implementation

The limiters are defined in `server/utils/ratelimit.ts` and applied in:

- `server/utils/tmdb.ts` before outbound TMDB requests (`tmdbLimiter`)
- `server/utils/gemini.ts` for authenticated recommendation generation (`recommendationLimiter`)

```ts
// server/utils/ratelimit.ts
const tmdbLimiter = new Ratelimit({
  redis,
  limiter: Ratelimit.slidingWindow(40, '1 s'),
  analytics: true,
})
```

## Environment Setup

Two Upstash Redis environment variables are required:

```bash
UPSTASH_REDIS_REST_URL=https://your-db.upstash.io
UPSTASH_REDIS_REST_TOKEN=your_token_here
```

Get these from [Upstash console](https://console.upstash.com) after creating a Redis database.
