The TMDB Proxy allows the frontend to securely request data from The Movie Database API without exposing the API key to the client.

## Endpoint

```text
GET /api/tmdb/:path
```

## How It Works

1. Client sends request to `/api/tmdb/...`
2. Server receives the request and extracts the path
3. Server adds `NUXT_TMDB_API_KEY` from environment variables
4. Request is forwarded to TMDB API with all query parameters
5. Response is returned to client

## Example Usage

### Get Movie Details

For single movies, prefer `GET /api/movies/:id` â€” it adds 3-month server-side caching and returns a normalised shape (`poster`, `duration`, `actors`, `trailer`). Use the TMDB proxy only for raw data not covered by that endpoint.

::code-group
```js [Cached (preferred)]
const response = await fetch('/api/movies/550')
const movie = await response.json()
```

```js [TMDB Proxy (raw)]
const raw = await fetch('/api/tmdb/movie/550')
const rawMovie = await raw.json()
```
::

### Search Movies

```js
const query = 'The Matrix'
const response = await fetch(`/api/tmdb/search/movie?query=${encodeURIComponent(query)}`)
const results = await response.json()
```

## Request Format

### Path Parameter

- **path** - The TMDB API path (e.g., `movie/550`, `search/movie`)

### Query Parameters

Pass any TMDB API parameters as query strings:

- `query` - Search string
- `page` - Results page number
- `sort_by` - Sort order
- `with_genres` - Genre filter
- etc. (all TMDB parameters supported)

### Headers

The following headers are automatically set by the proxy:

```text
Accept: application/json
Authorization: Bearer [from NUXT_TMDB_API_KEY env]
language: en-US
```

## Response Format

Returns whatever TMDB API returns in JSON format.

**Example Movie Response:**

```json
{
    "id": 284053,
    "title": "Thor: Ragnarok",
    "poster": "/path/to/poster.jpg",
    "rating": 7.6,
    "year": 2017,
    "duration": "2h 11m",
    "runtime": 131,
    "genres": [...],
    "actors": [...],
    "directors": [...],
    "description": "Thor is imprisoned...",
    "trailer": "/path/to/yout"
}
```

## Error Handling

If TMDB API returns an error, the proxy forwards the error response:

```json
{
  "error": "Invalid API key",
  "statusCode": 401
}
```

## Environment Setup

Make sure to set the TMDB API key:

```bash
# .env
NUXT_TMDB_API_KEY=your_api_key_here
```

Get your key from [themoviedb.org/settings/api](https://www.themoviedb.org/settings/api)

## Rate Limiting

Requests are rate-limited **per client IP** â€” 40 requests per second sliding window. Every response includes headers so clients can track remaining capacity:

| Header                  | Description                                |
| ----------------------- | ------------------------------------------ |
| `X-RateLimit-Limit`     | Total allowed requests in the window       |
| `X-RateLimit-Remaining` | Requests left in the current window        |
| `X-RateLimit-Reset`     | Unix timestamp (ms) when the window resets |

When the limit is exceeded the server returns `429 Too Many Requests`.

See [Rate Limiting](/reference/rate-limiting) for full details.

## Security Notes

- API key is never exposed to the client
- Requests are handled server-side and forwarded to TMDB
- All queries are forwarded to TMDB
- Language is always set to English (configurable if needed)
