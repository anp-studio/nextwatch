# API Reference

The Movie Recommender system exposes several APIs for different use cases:

- **TMDB Proxy** - Forward requests to The Movie Database
- **Public API** - Access recommendations without authentication  
- **User API** - Access personalized data with user authentication
- **Admin API** - Manage system data and configuration

All APIs are served from `/api/` endpoint in the frontend server.

## Quick Reference

| API | Base URL | Auth | Rate Limit |
|-----|----------|------|-----------|
| TMDB Proxy | `/api/tmdb/*` | API Key (env) | TMDB limits |
| Public | `/api/public/*` | None | 100/min per IP |
| User | `/api/user/*` | Supabase Auth | Standard |
| Admin | `/api/admin/*` | Admin Token | None |

## Environment Variables

Required configuration for API functionality:

```bash
# TMDB API
NUXT_TMDB_API_KEY=your_tmdb_api_key_here

# Admin Protection
ADMIN_API_TOKEN=secure_token_here
``

## Authentication Methods

### API Key (TMDB Proxy)
Automatically injected from `NUXT_TMDB_API_KEY` environment variable.

### Admin Token
Include in request headers:
```bash
curl -H "X-Admin-Token: your_token_here" https://api.example.com/api/admin/...
```

### User Auth (Supabase)
Authenticated via Supabase session cookies or authorization headers.

## CORS & Security

- **Rate Limiting**: Public endpoints limited to 100 requests per minute per IP
- **Admin Protection**: Admin endpoints require valid `X-Admin-Token` header
- **User Protection**: User endpoints require Supabase authentication
- **TMDB Proxy**: Securely forwards requests with API key from server
