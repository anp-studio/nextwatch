# Backend Server

The `frontend/server/` folder contains the Nuxt server-side logic that connects the frontend application with external APIs and manages logic. It's built using Nuxt's server capabilities (powered by h3).

## Architecture Overview

The server is organized into four main sections:

```
frontend/server/
├── api/          # HTTP endpoints
├── middleware/   # Request processing
├── services/     # Logic
└── utils/        # Helper functions
```

## API Routes (`api/`)

### TMDB Proxy
**File:** `tmdb/[...path].ts`

An endpoint that forwards requests to The Movie Database (TMDB) API. It automatically includes authentication and normalizes responses.

- **Purpose:** Centralize TMDB API calls and manage credentials securely
- **Endpoint:** `/api/tmdb/*`
- **Key Features:**
  - Adds API key from environment variables
  - Sets default language to `en-US`
  - Handles JSON responses
  - Error handling with status code forwarding

### Public Similarities
**File:** `public/similarities/[movieId].get.ts`

Public endpoint to fetch similar movies and TV shows for any given movie.

- **Endpoint:** `/api/public/similarities/:movieId`
- **Query Parameters:**
  - `limit` - Maximum number of results (optional)
  - `minScore` - Minimum similarity score filter (optional)
- **Rate Limited:** Yes (100 requests per minute per IP)
- **Authentication:** None required

### User Similarities
**File:** `user/similarities/[movieId].get.ts`

User-authenticated endpoint for personalized recommendations.

- **Endpoint:** `/api/user/similarities/:movieId`
- **Authentication:** Required (via middleware)

### Admin Upload
**File:** `admin/upload-similarities.post.ts`

Allows authenticated admins to upload new similarity data.

- **Endpoint:** `/api/admin/upload-similarities`
- **Authentication:** Requires `X-Admin-Token` header
- **Method:** POST

## Middleware (`middleware/`)

Middleware functions process requests before they reach route handlers.

### Admin Authentication
**File:** `adminAuth.ts`

Validates admin API token for protected endpoints.

- **Protects:** All `/api/admin/*` routes
- **Mechanism:** Checks `X-Admin-Token` header against `ADMIN_API_TOKEN` env variable
- **Response:** 403 Forbidden if token is invalid or missing

### Rate Limiting
**File:** `rateLimit.ts`

Protects public endpoints from abuse.

- **Applies to:** `/api/public/*` routes
- **Limit:** 100 requests per minute per IP address
- **Response:** Sets rate limit headers in response

### User Authentication
**File:** `userAuth.ts`

Validates user sessions for protected routes.

- **Protects:** `/api/user/*` routes
- **Integration:** Works with Supabase auth

## Services (`services/`)

### Similarity Service
**File:** `similarity-service.ts`

Core logic for fetching and transforming similarity data.

**Main Function:** `getSimilaritiesById(movieId, options?)`

```ts
interface Options {
  limit?: number        // Max results to return
  minScore?: number     // Filter by minimum score (0-100)
}
```

**Returns:**
```ts
{
  movieId: number
  count: number
  similarities: Array<{
    id: number
    type: 'movie' | 'show'
    score: number
  }>
}
```

**Features:**
- Sorts results by score (highest first)
- Filters by minimum score
- Limits result count
- Transforms numeric type IDs to readable strings

## Utilities (`utils/`)

### Similarity Cache
**File:** `similarity-cache.ts`

In-memory caching system for pre-computed similarities.

**Key Features:**
- **Cache Duration:** 24 hours
- **Data Source:** CSV format similarity data
- **Parsing:** Converts CSV to Map structure

**Main Function:** `getSimilaritiesForId(movieId)`
- Returns cached similarity array for a movie
- Returns `null` if movie not found

### Storage
**File:** `storage.ts`

Handles downloading similarity data from external storage (S3 or similar).

**Main Function:** `downloadSimilarities()`
- Fetches CSV file from configured storage
- Returns Buffer of CSV data

## Environment Variables

Required environment variables for the server:

```env
NUXT_TMDB_API_KEY=your_key_here
ADMIN_API_TOKEN=secure_token_here
```

## Request Flow Example

**Fetching similarities for a movie:**

1. Client requests `/api/public/similarities/550`
2. Rate limit middleware checks IP quota
3. Route handler calls `getSimilaritiesById(550)`
4. Service queries cached similarities from `similarity-cache`
5. If cache is stale, `storage.ts` downloads fresh CSV
6. Similarities are filtered, sorted, and returned

## Security Model

| Route | Authentication | Rate Limit |
|-------|---|---|
| `/api/public/*` | None | 100/min |
| `/api/user/*` | Supabase Auth | Standard |
| `/api/admin/*` | Admin Token | None |
| `/api/tmdb/*` | API Key (env) | TMDB limits |
