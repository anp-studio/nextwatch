# Watched Movies API

`/api/watched` manages each authenticated user's watched movie list in Supabase.

::callout{type="info"}
This route supports `GET`, `POST`, and `DELETE` so the client can keep local and remote watched state in sync.
::

## Auth

- Requires `Authorization: Bearer <supabase_access_token>`
- User is resolved server-side from the token
- Requests without a valid token return `401 Unauthorized`

## Data Source

Table: `watched_movies`

| Column | Type | Description |
| --- | --- | --- |
| `id` | `uuid` | Row primary key |
| `user_id` | `uuid` | Owner user ID |
| `movies` | `jsonb` | Array of watched movie objects |
| `updated_at` | `timestamptz` | Last write timestamp |

Movie object shape:

- Required: `tmdbId`, `title`, `year`, `posterPath`
- Optional: `genres`, `runtime`

## Endpoints

### `GET /api/watched`

Returns the current user's watched list.

#### Response

```json
{
  "success": true,
  "movies": [
    { "tmdbId": 550, "title": "Fight Club", "year": 1999 }
  ]
}
```

### `POST /api/watched`

Adds a movie if it is not already present (`tmdbId` dedupe).

#### Request body

```json
{
  "movie": {
    "tmdbId": 550,
    "title": "Fight Club",
    "year": 1999,
    "posterPath": "/path.jpg"
  }
}
```

#### Response

```json
{
  "success": true,
  "watchedCount": 1
}
```

### `DELETE /api/watched`

Removes a movie from the watched list by `tmdbId`.

#### Request body

```json
{
  "tmdbId": 550
}
```

#### Response

```json
{
  "success": true,
  "watchedCount": 0
}
```

## Error Codes

- `400` invalid payload or invalid `tmdbId`
- `401` missing or invalid bearer token
- `404` no watched row exists when deleting
- `405` unsupported method
- `500` Supabase query/update/insert failure
